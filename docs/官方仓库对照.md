## 上游模块速览

| 目录 | 作用 | 对鸿蒙端的意义 |
|---|---|---|
| `cli/` | CLI 二进制，包装 agent | 鸿蒙端不依赖 |
| `hub/` | HTTP API + SSE + Socket.IO + Telegram | **对接目标**，鸿蒙端只消费其 API |
| `web/` | React PWA 前端 | **移植来源**，逐页面 / 组件对照 |
| `shared/` | 类型、Zod schema、事件定义 | 类型对齐基准 |
| `docs/` | VitePress 文档 | 行为参考 |
| `website/` | 营销站 | 不相关 |

## 移植对照表

### 网络 / API

| 上游 | 鸿蒙端 | 备注 |
|---|---|---|
| `web/src/api/client.ts` | `api/HubClient.ts` | REST 封装 |
| `web/src/hooks/useSSE.ts` | `api/SSEClient.ts` | SSE 等价 |
| `web/src/hooks/queries/` | `models/` + 页面 `@State` | TanStack Query → ArkUI 状态 |

### 页面

| 上游路由 | 鸿蒙页面 | 说明 |
|---|---|---|
| 会话列表 | `pages/SessionListPage` | 含状态指示（attention + scheduled） |
| 聊天 | `pages/SessionChatPage` | 消息流 + 权限审批 |
| 文件 | `pages/FilesPage` | 文件树浏览 |
| 终端 | `pages/TerminalPage` | xterm.js → 鸿蒙终端组件 |

### 类型

| 上游 | 鸿蒙端 |
|---|---|
| `shared/src/types.ts`（Session / Message / Machine） | `models/types.ts` |
| `shared/src/schemas.ts`（Zod） | ArkTS 类型 + 手写校验 |
| `shared/src/socket.ts` | 仅参考事件名；鸿蒙端走 SSE / REST |

## Hub API 速查（对接目标）

摘自 `参考源码/hapi/hub/README.md`。

### 配置

| 项 | 默认 | 说明 |
|---|---|---|
| `CLI_API_TOKEN` | 首次自动生成 | CLI 与 Web/App 登录共享密钥；客户端追加 `:namespace` 隔离 |
| `HAPI_LISTEN_PORT` | `3006` | HTTP 端口 |
| `HAPI_LISTEN_HOST` | `127.0.0.1` | 远程访问需改 / 反代 |
| `HAPI_PUBLIC_URL` | — | 公网 HTTPS；Telegram Mini App 与默认 CORS |
| `CORS_ORIGINS` | 由 PUBLIC_URL 推导 | 鸿蒙端直连需放行 |

### 通道

- **REST API**：会话、消息、权限、机器、文件
- **SSE**：`/api/events` 实时更新
- **Socket.IO**：CLI 连接通道（鸿蒙端不直连）

### 启动

```bash
npx @twsxtd/hapi hub --relay    # E2E 加密中继（WireGuard + TLS）
npx @twsxtd/hapi                # 跑 claude code（CLI 端）
```

## 同步上游

```bash
cd 参考源码/hapi
git fetch origin
git log --oneline -10 origin/main
git checkout main && git pull
git rev-parse HEAD              # 取新 commit，回填本文件与 README 的表格
```

> 关注点：上游 `shared/src` 类型与 `hub/` API 变更需及时跟进；`web/` 视觉变更按需移植。
