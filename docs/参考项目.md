# 参考项目

移植 Hapi 鸿蒙客户端时，有两套鸿蒙侧参考。**优先级：chatcube 为主，berry3 为辅。**

## 参考优先级

| 优先级 | 项目 | 参考范围 |
|---|---|---|
| 🥇 **主要** | chatcube（Cube Chat） | **布局、样式、流式对话、对话、会话管理** —— 业务 / 架构 / UX 全方位对齐 |
| 🥈 **次要** | berry3（Berry Browser） | 只做**底层代码参考**（系统能力调用、特定底层实现） |
| 语义 | `参考源码/hapi`（官方上游） | 功能行为、Hub 交互协议、数据结构（见 [官方仓库对照.md](官方仓库对照.md)） |

> **为什么这样分**：chatcube 是 ArkTS 原生 AI 聊天客户端，与 Hapi 客户端业务同构（都是 AI 对话 + 会话管理），作为主要参考；berry3 虽是成熟商业 App，但业务是浏览器，与 Hapi 不同，**仅取其底层 / 系统能力的实现写法**。

## chatcube（Cube Chat）—— 主要参考

面向鸿蒙 6（API 23）的**原生 AI 聊天客户端**：ArkTS 原生开发（非 WebView），15+ provider，联网搜索 + MCP，MVVM 架构。与 Hapi 客户端几乎同构，是布局 / 样式 / 对话 / 会话的一手参考。

| bundleName | `com.longlive.chatcube` |
| 版本 | `1.3.1` |
| Git 仓库 | https://github.com/LongLiveY96/chatcube |
| 本地路径 | `参考源码/chatcube` |
| 当前 commit | `fd729d0` |
| 状态 | **已归档（只读快照）** |
| License | **MIT** |

### 页面 / 能力参考映射

| Hapi 要做的 | chatcube 参考 |
|---|---|
| **登录页**（填 Hub 地址 + token） | `components/onboarding/OnboardingStepProvider`、`components/provider`、`components/settings`（provider 配置范式，正好对应「填地址 + key」） |
| **会话列表页** | `pages/index/`：`IndexSessionListDataSource`（列表数据源）、`IndexSessionGroupHelper`（分组）、`IndexTabsVisibilityHelper`（标签）、`IndexSettingsSheetCoordinator`（设置面板） |
| **对话页** | `pages/chat/`：`ChatSendFlow`（发送）、`ChatStopFlow`（停止）、`ChatScrollUiFlow`（滚动）、`ChatReasoningStateHelper`（推理状态）、`ChatToolIdHelper`（工具调用）、`ChatInlineEditFlow`（内联编辑）、`ChatSelectionFlow`（选择） + `components/chat/` |
| **流式对话** | `pages/chat` 的 Flow 系列（发送 / 停止 / 滚动 / 推理 均为流式驱动）+ `services/`（SSE / 流式协议） |
| **会话管理** | `pages/index` + `state/`（会话状态）+ `viewmodels/` |

### 架构要点（可借鉴）

- **MVVM**：`viewmodels/`（视图模型）+ `state/`（含 `migrations/` 状态迁移）+ `models/`
- **provider 注册**：`services/registry/`（多服务商统一接入，类比 Hapi 接不同 Hub）
- **设计系统**：`ui/hds/`（自有 UI 设计系统）
- **联网搜索 / MCP**：`services/search/`

## berry3（Berry Browser）—— 次要参考（仅底层代码）

成熟的鸿蒙原生浏览器 App。业务（浏览器）与 Hapi 不同，**不作为布局 / 样式 / 业务参考**；只在需要**底层 / 系统能力**实现时去翻。

| License | 未明确 |

### 什么时候用 berry3

只在这些场景翻 berry3：

- 某**系统能力**（通知、后台任务、文件、网络底层、窗口管理…）在 chatcube 里找不到现成实现
- chatcube 没覆盖的**底层写法**（如 NAPI / cpp 交互、特定系统 API）

> 规则：**先 chatcube，chatcube 没有再 berry3。** 布局 / 样式 / 对话 / 会话一律以 chatcube 为准，不要参考 berry3 的浏览器 UI。

### berry3 附带的鸿蒙 skills

berry3 自带一整套 skills（在 `berry3/.claude/` 下），其中「查询鸿蒙官方文档」的两个 retriever **已复制到本项目**（见下）。其余（崩溃分析、打包、模拟器等）随用随到 berry3 翻。

## 鸿蒙官方文档检索（skill，本项目自带）

查询鸿蒙官方文档的两个 retriever skill，自包含、纯 Python 标准库、已复制到 `.claude/skills/`：

| skill | 用途 |
|---|---|
| `hmos-arkts-knowledge-retriever` | 检索 **ArkTS 语言指南**（语法、库、并发、运行时） |
| `hmos-arkui-knowledge-retriever` | 检索 **ArkUI 知识库**（组件、API、状态管理 V1/V2、错误码） |

由 AI 助手按需触发。脚本入口：`scripts/search_docs.py`（arkts）、`scripts/run.py`（arkui），需 Python 3。
